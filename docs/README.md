## üìä Documentation Standards

### Diagrams
- Use ASCII art for simple flows
- Use Mermaid for complex diagrams (future)
- Keep diagrams up-to-date with code

### Code Examples
- Use TypeScript for code snippets
- Include comments for clarity
- Show both correct and incorrect examples

### Formatting
- Use Markdown for all documentation
- Follow consistent heading hierarchy
- Use tables for structured data
- Include code blocks with syntax highlighting
